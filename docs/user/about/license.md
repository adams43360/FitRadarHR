## Ce que ça signifie concrètement

✅ Vous pouvez consulter, cloner et étudier le code librement

✅ Vous pouvez l'utiliser à des fins de recherche ou d'éducation non commerciales
✅ Vous pouvez l'utiliser dans le cadre d'une prestation de service pour un client qui l'utilise lui-même sous ces mêmes conditions
✅ Deux ans après sa publication, chaque version repasse automatiquement sous licence **MIT** — sans aucune démarche de votre part

❌ Vous ne pouvez pas, sans autorisation, l'utiliser pour proposer à des tiers un produit ou service commercial qui concurrence FitRadarHR — c'est-à-dire qui s'y substitue, qui concurrence une offre que nous proposons déjà, ou qui offre une fonctionnalité substantiellement similaire

La seule contrainte sur les usages autorisés : conserver la mention de copyright et un lien vers la licence dans les copies redistribuées.

**En résumé** : auto-héberger FitRadarHR pour votre RH ou vos équipes reste entièrement libre et gratuit. Ce qui est réservé, c'est le fait d'en faire un produit ou service commercial concurrent sans notre accord.

## Pourquoi ce n'est plus "open source" au sens strict

La définition officielle de l'open source (Open Source Initiative) interdit toute restriction sur l'usage commercial. Une licence qui réserve un usage — même seulement l'usage commercial concurrent — n'est donc plus "open source" au sens OSI, même si le code reste public et lisible. C'est pour ça qu'on parle de **"Fair Source"** plutôt que d'"open source" : le code est ouvert, mais l'usage n'est pas sans aucune limite. D'autres projets connus utilisent ce même modèle : Sentry, HashiCorp (avant son rachat), CockroachDB.
